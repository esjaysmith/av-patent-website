# File and Directory Naming Conventions

## General Rules

- Use lowercase letters only
- Use hyphens (-) to separate words
- Avoid spaces, underscores, or special characters
- Keep names descriptive but concise
- Use English language for all names

## Directory Structure

- `assets/` - Static assets
  - `css/` - Stylesheets
  - `js/` - JavaScript files
  - `images/` - Image files
  - `documents/` - PDF and document files
  - `media/` - Video and audio files

- `pages/` - HTML pages
  - `core/` - Main website pages
  - `blog/` - Blog articles
  - `landing/` - Landing pages

- `templates/` - Reusable HTML templates
- `config/` - Configuration files
- `docs/` - Documentation
- `deployment/` - Deployment scripts

## File Naming Examples

### HTML Files
- `index.html` - Homepage
- `patents-overview.html` - Patents overview page
- `market-analysis.html` - Market analysis page
- `tesla-fsd-v13-analysis.html` - Specific landing page

### CSS Files
- `main.css` - Main stylesheet
- `responsive.css` - Responsive styles
- `components.css` - Component styles

### JavaScript Files
- `main.js` - Main JavaScript file
- `form-validation.js` - Form validation script
- `analytics.js` - Analytics tracking

### Image Files
- `hero-background.jpg` - Hero section background
- `patent-diagram.png` - Patent illustration
- `tesla-logo.svg` - Company logo

### Document Files
- `patent-summary.pdf` - Patent summary document
- `market-report.pdf` - Market analysis report

## URL Structure

- Use clean URLs matching file names
- Example: `/tesla-fsd-v13-analysis` for `tesla-fsd-v13-analysis.html`

## Version Control

- Commit messages should be descriptive
- Use present tense for commit messages
- Group related changes in single commits
