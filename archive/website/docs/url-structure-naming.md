# URL Structure and Naming Conventions

## URL Structure Guidelines

### Base URL
- Domain: `https://autonomous-patent-licensing.com` (placeholder)
- All URLs relative to root: `/page-name`

### URL Patterns

1. **Homepage**
   - URL: `/`
   - Purpose: Main entry point

2. **Core Pages**
   - Pattern: `/{page-name}`
   - Examples: `/patents-overview`, `/market-analysis`
   - Length: 2-4 words maximum

3. **Landing Pages**
   - Pattern: `/{primary-keyword}-{secondary-keyword}-{focus}`
   - Examples: `/tesla-fsd-camera-revolution`, `/robotaxi-market-2034-forecast`
   - Length: 3-6 words, include main keywords

4. **Category Pages** (if needed)
   - Pattern: `/category/{subcategory}`
   - Example: `/blog/tesla-fsd-updates`

## Naming Conventions

### General Rules
- All lowercase letters
- Hyphen-separated words (kebab-case)
- No spaces, underscores, or special characters
- English language only
- Descriptive and keyword-rich
- Maximum 50 characters per URL

### Keyword Integration
- Include primary keyword early in URL
- Add secondary keywords naturally
- Avoid keyword stuffing
- Match search intent

## SEO Considerations

### URL Length
- Ideal: 50-60 characters
- Maximum: 75 characters
- Avoid overly long URLs

### Keyword Optimization
- Primary keyword in first position
- Natural keyword flow
- User-friendly readability

### Technical Requirements
- Static URLs (no query parameters)
- Clean, hackable URLs
- Consistent structure across site
